SamQuicksal

About

SamQuicksal is a capstone project for IT Web and Mobile Application. Its primarily purpose is to create a web and mobile application system that allows user to locate nearby samgyeopsal restaurant, book for reservation, queue in line using the app, order menu dishes, do payment within the app, give rating and enjoy rewards and promotions. Please note that this is only the mobile app design of SamQuicksal.

Problem Statement

Customers of Samgyeopsal restaurants are unable to wait for long queue lines and are unable to order because of shyness or a lack of confidence when raising their hand upon calling for a waiter.

Objective

To help customers know their queue while not patiently waiting in line and also to help customers order their preferred dish by using the application.

Takeaways

Summary

Ordering have inevitable problems like Order-taking problems, this can cause delay and can affect with the number of customers coming in the restaurant. We have created a feature that solves this user problem, this feature helps user select a dish from the given menu.

We have also added features that can help users and benefit restaurant owners and staffs or the restaurant itself. These following are the features we have added.

  • Developed a reservation module capable of reserving for a specific number of customers
  • Developed a location module that will recommend the nearest samgyeopsal restaurants based on the user's location
  • Developed a notification module that will notify the user when their table is ready through SMS/Push notifications
  • Integrated a payment module using GCash App
  • Implemented a customer loyalty rewards module, providing customers with discounts and vouchers.

Challenges

We don't have a client for the capstone project, so we considered all scenarios that can happen in different samgyeopsal restaurants. From considering whether they have a time limit or not, and if they offer reservation or queueing only.

Lessons Learned

  • Don't let users keep them wondering what will happen next. Provide feedback in every action they make.
  • Avoid biases. Focus on user research and do not base on self experience as this can affect the team's decision making.
  • I should keep the project well documented and update as often as possible as it can reduce the time of finding the specific feature within the app for iteration purposes.
